## 4. psql 验证 v1.1 关键加固 (10 分钟)

> 这一节是答辩时最有讲点的: 直接在 psql 里 "扮演恶意运维" 试图绕过 controller, 看 DB 层防御是否生效。

### 4.1 B2 · 进货状态机非法跳变拒绝

**[终端 A]**

```powershell
psql -h 127.0.0.1 -U postgres -d booknook
```

进入 psql 交互后, 一行一行粘贴:

```sql
BEGIN;
-- 创建一张 pending 进货单
INSERT INTO purchase_orders (order_no, created_by, supplier, status, total_amount)
VALUES (gen_order_no('PO'), 1, 'test', 'pending', 100.00)
RETURNING id, status;
-- 接下来试图直接跳到 received (绕过 paid)
UPDATE purchase_orders SET status='received'
 WHERE id = (SELECT MAX(id) FROM purchase_orders) AND status='pending';
ROLLBACK;
\q
```

`✅ 预期`: UPDATE 语句被触发器拦截, 报错:
```
错误:  非法的进货单状态变更: pending -> received
提示:  合法转移仅有 pending->paid / pending->returned / paid->received
```

`📣 答辩话术`: "**触发器在 ELSE 分支显式 RAISE EXCEPTION, 即便有人直接在 psql 里改状态也会被拒绝, 这是把业务规则真正下沉到数据库层。**"

### 4.2 B1 · 并发取号 (advisory_xact_lock)

**[终端 A]**

```powershell
psql -h 127.0.0.1 -U postgres -d booknook -c "SELECT gen_order_no('PO') AS a, gen_order_no('SO') AS b;"
```

`✅ 预期`:
```
        a         |        b
------------------+------------------
 PO-20260519-0003 | SO-20260519-0003
```

注意日期是当天, 格式 `PO-YYYYMMDD-XXXX`。两次调用同事务内会复用 advisory 锁, **如果两个并发事务调用, 一个会等另一个提交后才能拿到不同的 seq**, 不会再产生 UNIQUE 冲突。

如果想模拟并发, 开两个 PSQL 终端各跑:

**[终端 A]**:
```sql
BEGIN;
SELECT gen_order_no('PO');
-- 故意不 COMMIT, 让锁继续持有
```

**[另一个终端]**:
```sql
SELECT gen_order_no('PO');
-- 这里会阻塞直到 [终端 A] COMMIT/ROLLBACK
```

`✅ 预期`: 第二个终端的 `SELECT` 一直等待, 直到 [终端 A] `COMMIT` 后才返回, 而且返回的 seq 是 +1 之后的值。

回到 [终端 A] 输 `COMMIT;` 后两端都解锁。

### 4.3 B4 · 入库 ISBN 元数据一致性

**[终端 A]**

```sql
-- 先看一本已有书的 ISBN
psql -h 127.0.0.1 -U postgres -d booknook -c "SELECT isbn, title FROM books LIMIT 1;"
```

记下 isbn 和 title。然后试图录入一个 "ISBN 已存在, 但 title 不同" 的新书进货单:

```sql
psql -h 127.0.0.1 -U postgres -d booknook
```

```sql
BEGIN;
-- 用一个已有 ISBN 但故意写错 title
INSERT INTO purchase_orders (order_no, created_by, supplier, status, total_amount)
VALUES (gen_order_no('PO'), 1, 'test', 'pending', 100)
RETURNING id;

-- 假设上面拿到 id=N, 把下面的 N 改成实际值
INSERT INTO purchase_order_items (order_id, isbn, title, publisher, author, purchase_price, quantity)
VALUES ((SELECT MAX(id) FROM purchase_orders),
        '9787020002207',     -- 这是《红楼梦》的 ISBN
        '《盗版红楼梦》',     -- 但 title 故意写错
        '盗版出版社', '盗版作者', 30.00, 5);

-- 推进到 paid
UPDATE purchase_orders SET status='paid' WHERE id=(SELECT MAX(id) FROM purchase_orders);

-- 推进到 received → 触发器应该拒绝
UPDATE purchase_orders SET status='received' WHERE id=(SELECT MAX(id) FROM purchase_orders);

ROLLBACK;
\q
```

`✅ 预期`: 最后一条 UPDATE 报错:
```
错误:  ISBN 9787020002207 已存在但元数据不一致: 库内《红楼梦》(曹雪芹, 人民文学出版社) vs 录入《《盗版红楼梦》》(盗版作者, 盗版出版社)
提示:  请改用已有书的 book_id, 或核对录入的书名/作者/出版社
```

`📣 答辩话术`: "**入库时如果 ISBN 命中老书但 title/author/publisher 不一致, 触发器拒绝, 防止 ' 老书改名 ' 这种静默数据污染。**"

### 4.7 C4 · 登录速率限制

**[终端 A]** (用一个一次性 cookie jar 避免影响后续步骤)

```powershell
# 连续打 25 次错误密码
1..25 | ForEach-Object {
  $r = curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:4000/api/auth/login `
       -H 'Content-Type: application/json' `
       -d '{\"username\":\"super\",\"password\":\"wrong\"}'
  "$_`: $r"
}
```

`✅ 预期`: 前 20 次返回 `401`, 第 21 次开始变成 `429` (速率限制触发)。

> 等 15 分钟后限速会自动重置, 或者重启后端 (`npm run dev` 终端 Ctrl+C 再启动) 立即清空内存计数。

`📣 答辩话术`: "**登录限速 20 次/15min/IP, 防字典攻击。MD5 校验本来很快, 攻击者每秒能跑上万次, 限速后被指数级抬高破解成本。**"

```powershell
$ErrorActionPreference = 'Stop'

$check = psql -h 127.0.0.1 -U postgres -d booknook -A -t -c "
SELECT
  (SELECT COALESCE(SUM(total_amount),0) FROM sale_orders) =
  (SELECT COALESCE(SUM(total_amount),0) FROM v_daily_sales_trend) AS a1,
  (SELECT bool_and(so.total_amount = (SELECT SUM(subtotal) FROM sale_order_items WHERE order_id=so.id))
     FROM sale_orders so) AS a2,
  (SELECT bool_and(tx.amount = (SELECT SUM(subtotal) FROM sale_order_items WHERE order_id=tx.related_id))
     FROM transactions tx WHERE tx.related_table='sale_orders') AS b5;
"
if ($check -ne 't|t|t') {
  Write-Host "✗ 一致性检查失败: $check" -ForegroundColor Red

if (-not $ready) {

python .\scripts\smoke-test.py
$exit = $LASTEXITCODE

Write-Host "停止后端..."
Stop-Process -Id $backend.Id -Force -ErrorAction SilentlyContinue
Get-NetTCPConnection -LocalPort 4000 -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force -ErrorAction SilentlyContinue }

if ($exit -eq 0) {
  Write-Host ""
  Write-Host "✅ 全套验证通过! 可以开始 §5 前端手工验证了。" -ForegroundColor Green
```
