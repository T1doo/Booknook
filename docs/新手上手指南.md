## 项目长啥样? (1 分钟脑补)

BookNook 是一个 **图书销售管理系统**, 跑起来后浏览器里你能看到这些功能:

```
登录页 (棕色书架风背景, 输入 super / Admin@2026)
   ↓
左侧栏菜单 (10 个功能):
   ├─ 概览 Dashboard    → 4 张 KPI 卡 + 4 个图表
   ├─ 库存图书           → 30 本预置的书, 可搜索/编辑
   ├─ 进货管理           → 新建进货单 → 付款 → 入库
   ├─ 销售收银           → 像超市收银台一样, 加购物车 → 结账
   ├─ 财务账单           → 所有收入支出流水
   ├─ 报表导出           → 一键下载 Excel/PDF
   ├─ 库存预警           → 库存不足自动提醒
   ├─ 用户管理 (超管)    → 创建管理员账号
   ├─ 操作日志 (超管)    → 所有改动可追溯
   └─ 我的信息           → 改密码 / 改资料
```

整套东西由 **三个部分** 组成, 必须**同时启动**才能用:

```
浏览器 (你)                ← 你看到的界面
   ↕  HTTP
前端 (Next.js, 端口 3000)  ← 把数据渲染成漂亮页面
   ↕  HTTP
后端 (Node.js, 端口 4000)  ← 处理业务逻辑 + 鉴权
   ↕  SQL
数据库 (PostgreSQL)        ← 永久存储所有数据
```

**你要做的事**: 让这三个东西都跑起来。

## 第 1 章 · 装好你需要的工具 (20 分钟)

你要装 4 个工具。每个都给个一句话说明:

| 工具 | 是什么 | 为什么需要 |
|---|---|---|
| **Node.js** | JavaScript 运行环境 | 后端和前端都是 JS 写的, 需要它来跑 |
| **PostgreSQL** | 关系数据库 | 存所有书、订单、用户数据的地方 |
| **Python** | 一种编程语言 | 仅 smoke-test 脚本用, 可选 |
| **Git** | 版本控制工具 | 下载项目用; 如果已经下载过 zip 包可以跳过 |

### 1.1 检查你已经装了什么

打开 PowerShell, 一行一行粘:

```powershell
# [窗口 A]
node --version
npm --version
psql --version
python --version
git --version
```

每一条都会输出版本号或者报错。

`✅ 全装好的样子`:
```
v24.10.0           ← Node.js
10.9.3             ← npm (装 Node 时自带)
psql (PostgreSQL) 16.4
Python 3.12.4
git version 2.45.1
```

如果某条报 `不是内部或外部命令`, 那就是这个工具没装, 去对应小节装。

### 1.2 装 Node.js

> Node.js 是项目的"发动机"。装一次, 自带 `npm` (包管理器)。

**步骤**:
1. 浏览器打开 https://nodejs.org/
2. 点 "LTS 版本" 那个**绿色**按钮下载 (写着 "Recommended For Most Users")
3. 双击下载的 `.msi` 文件
4. **一路点 Next**, 中间有个 "Automatically install necessary tools" 复选框, **打勾** (它会装一些编译工具)
5. 装完, **关掉所有 PowerShell 窗口**, 重新开一个

验证:
```powershell
# [窗口 A · 重开的]
node --version
npm --version
```

`✅ 期望`: 都能输出版本号, Node 至少 v20。

### 1.3 装 PostgreSQL

> PostgreSQL 是数据库, 项目所有数据都存在它里面。这一步**最容易踩坑**, 注意。

**步骤**:
1. 浏览器打开 https://www.postgresql.org/download/windows/
2. 点 "Download the installer" → 跳到 EDB 网站
3. 选 **PostgreSQL 16.x** (任何小版本都行) → "Windows x86-64"
4. 下载 .exe 文件 (大概 350MB)
5. 双击运行, 进入安装向导:
   - Installation Directory: 默认 `C:\Program Files\PostgreSQL\16` 即可
   - Select Components: 全选 (PostgreSQL Server / pgAdmin 4 / Stack Builder / Command Line Tools)
   - Data Directory: 默认即可
   - **Password**: ⚠️ **这里设置 postgres 用户的密码, 一定要记住!** 比如 `060425` 或你自己设的。**这个密码后面要用很多次**。
   - Port: 默认 **5432**, 不要改
   - Locale: 默认即可
   - 一路 Next 装完 (大概 5 分钟)
6. 装完会弹一个 "Stack Builder"窗口, **直接 Cancel 跳过**, 不需要装额外的东西。

验证:
```powershell
# [窗口 A · 重开]
psql --version
```

`✅ 期望`: 输出 `psql (PostgreSQL) 16.x`。

`❌ 如果报错 "不是内部或外部命令"`: PostgreSQL 装了但 PATH 没加。手动加:
- 右键 "此电脑" → 属性 → 高级系统设置 → 环境变量
- 系统变量里找到 `Path` → 编辑 → 新建
- 添加 `C:\Program Files\PostgreSQL\16\bin`
- 确定保存, 重开 PowerShell 重新验证

### 1.4 装 Python (可选)

> Python 只用于跑 smoke-test 脚本验证后端 API。如果你不打算跑测试, 可以**跳过**。

**步骤**:
1. 浏览器打开 https://www.python.org/downloads/
2. 点黄色按钮下载最新版 (3.10+)
3. 双击安装:
   - ⚠️ 第一屏底部 **勾选 "Add python.exe to PATH"** (默认是不勾的, 容易忘)
   - 点 "Install Now"
4. 装完关掉 PowerShell 重开。

验证: `python --version` 看到 `Python 3.x.x` 即可。

### 1.5 装 Git (如果你打算 git clone)

> 如果你是**下载 zip 包**解压, 可以跳过 Git。如果你想 `git clone`, 需要装。

下载: https://git-scm.com/download/win → 默认下载 64-bit 版本

装的时候一路 Next 即可, 默认设置就行。

### 1.6 (可选) 装 VS Code

> 不是必须, 但强烈推荐, 用来看项目代码。

下载: https://code.visualstudio.com/ → 一路 Next。

装完打开它, 左侧 Extensions 搜索安装:
- `Chinese (Simplified) Language Pack` (中文界面)
- `Prisma` (ORM 高亮)
- `ESLint`

### ✅ 第 1 章结束自检

打开新的 PowerShell:

```powershell
# [窗口 A]
node --version    # 期望: v20+
npm --version     # 期望: 10+
psql --version    # 期望: PostgreSQL 16.x
python --version  # 可选: 3.10+
```

全部正常 → 进第 2 章。
任何一个报错 → 回到对应小节重装, 或看 [FAQ 1.x](#faq-1-工具安装相关)。

## 第 3 章 · 启动数据库 (10 分钟)

> 这一章你要做的事: 让数据库**记住**有 30 本书、3 个用户、2 张订单……这些预置数据。

### 3.1 检查 PostgreSQL 服务在运行

PostgreSQL 装好后会自动启动一个 Windows 服务, 一直在后台运行。先确认它在跑:

```powershell
# [窗口 A]
Get-Service postgresql*
```

`✅ 期望`:
```
Status   Name              DisplayName
------   ----              -----------
Running  postgresql-x64-16 postgresql-x64-16
```

状态是 `Running` 就行。

`❌ 如果是 Stopped`: 手动启动:
```powershell
Start-Service postgresql-x64-16
```

### 3.2 设置密码到环境变量

每次跑 `psql` 命令都要输密码很烦, 把密码塞到环境变量里, 这个 PowerShell 窗口内的命令就会自动用它:

```powershell
# [窗口 A]
$env:PGPASSWORD = "你装 PostgreSQL 时设的密码"
# 比如: $env:PGPASSWORD = "060425"
```

> ⚠️ 这只在**当前窗口**生效, 关掉窗口就没了。每次开新窗口要重新设。

测一下能不能连上:
```powershell
psql -h 127.0.0.1 -U postgres -d postgres -c "SELECT version();"
```

`✅ 期望`: 输出 `PostgreSQL 16.x on ...` 一行版本号。

`❌ 报 password authentication failed`:
密码不对。重新设环境变量, **注意大小写**。

### 3.3 跑一键初始化脚本

项目内有一个 PowerShell 脚本会:
1. 删除旧的 `booknook` 数据库 (如果有)
2. 创建一个新的 `booknook` 数据库
3. 跑 `01-schema.sql` 创建 10 张表 + 6 个触发器 + 6 个函数
4. 跑 `02-seed.sql` 灌入 3 个用户、30 本书、2 张销售单的种子数据
5. 跑 `03-views.sql` 创建 6 个 Dashboard 视图

```powershell

```

如果 PowerShell 拒绝执行脚本, 报 "无法加载文件 ... 因为在此系统上禁止运行脚本":
```powershell
# 临时给当前用户开个口子, 一次性命令:
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
# 输 Y 回车确认, 然后重新跑上面的脚本
```

`✅ 成功输出` 大概长这样 (会刷过去一长串, 看最后):

## 第 4 章 · 启动后端 (15 分钟)

> 这一章你要做的事: 让后端服务在 `http://localhost:4000` 跑起来, 并保持运行。

### 4.1 进入后端目录

```powershell
# [窗口 A]
cd E:\Github\Booknook\backend
```

### 4.2 装后端依赖 (首次)

后端用了很多第三方库 (Express、Prisma、JWT 等), 这些库在 `package.json` 里列着, 用 npm 一键装:

```powershell
npm install
```

> 💡 `npm install` 在干啥: 它读 `package.json` 列出的依赖, 然后从 npm 仓库下载每个库到 `node_modules\` 文件夹。第一次装大概 **1-3 分钟**, 网络慢可能更久。

`✅ 成功`: 最后看到类似
```
added 350 packages, and audited 351 packages in 2m
found 0 vulnerabilities  (或 N moderate vulnerabilities, 可以无视)
```

`❌ 卡在某一步几分钟没动`: 网络问题, Ctrl+C 取消后重试。或换淘宝镜像:
```powershell
npm config set registry https://registry.npmmirror.com
npm install
```

> 💡 装完会发现项目根下多了个 `node_modules\` 文件夹, 几百 MB, 这是正常的。

### 4.3 配置环境变量 `.env`

后端需要一个 `.env` 文件存配置 (数据库连接串、JWT 密钥等)。项目里已经有了 `backend\.env`, 但你需要**确认里面两件事正确**。

打开 `backend\.env` (用记事本或 VS Code):

```powershell
# [窗口 A]
notepad backend\.env
```

你应该看到类似:
```
PORT=4000
CORS_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
JWT_SECRET=<一长串十六进制>
JWT_EXPIRES_IN=7d
COOKIE_NAME=booknook_token
DATABASE_URL=postgresql://postgres:060425@127.0.0.1:5432/booknook?schema=public
LOG_LEVEL=info
NODE_ENV=development
```

#### 4.3.1 改 `DATABASE_URL` 里的密码

找到 `DATABASE_URL=...` 这一行, 把 `060425` 改成**你自己设的 postgres 密码**:

```
DATABASE_URL=postgresql://postgres:你的密码@127.0.0.1:5432/booknook?schema=public
```

如果密码里有特殊字符 (`@`、`#`、`/` 等), 需要 URL 编码, 看 [FAQ 4.1](#faq-4-后端启动相关)。

#### 4.3.2 检查 `JWT_SECRET` 不是默认占位

后端启动时会**拒绝**使用模板默认值, 如果你看到:
```
JWT_SECRET=please-change-this-...
JWT_SECRET=<RUN-THE-COMMAND-ABOVE-AND-PASTE-HERE>
JWT_SECRET=do-not-use-in-production
```

需要**重新生成一个**:

```powershell
# [窗口 A]
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

输出一长串看起来像乱码的字符 (96 个十六进制), 比如:
```
f3e2e69cd224b733d1cf7475fad698aace57098097d5f0a1d445a067526563facd4c41453e638bc2ee4bad8a995b4718
```

**复制这串**, 回到 notepad 里, 把 `JWT_SECRET=` 后面的内容替换成它:
```
JWT_SECRET=f3e2e69cd224b733d1cf7475fad698aace57098097d5f0a1d445a067526563facd4c41453e638bc2ee4bad8a995b4718
```

保存 (Ctrl+S) 关闭记事本。

## 常见问题 FAQ

### FAQ 1 · 工具安装相关

**Q: `node --version` 报 "不是内部或外部命令"**
A: Node.js 没装, 或者装完没重开 PowerShell。回去重装, 装完关掉**所有** PowerShell 重开。

**Q: `psql --version` 报错, 但我装了 PostgreSQL**
A: PATH 没加。手动加 `C:\Program Files\PostgreSQL\16\bin` 到系统环境变量 Path 里, 重开 PowerShell。

**Q: 我忘了 postgres 密码**
A:
1. 用 pgAdmin (装 PostgreSQL 时一起装的 GUI 工具) 进去改
2. 或者重装 PostgreSQL 选择新密码 (会清掉所有现有数据)

**Q: 我能不能用 cmd 而不是 PowerShell?**
A: 可以, 但 `$env:VAR = "x"` 这种 PowerShell 语法在 cmd 里要换成 `set VAR=x`。本指南命令都是 PowerShell 风格, 用 cmd 要自己换。

---

### FAQ 2 · 项目代码相关

**Q: `npm install` 卡住或者报很多 ERR**
A:
1. 网络问题, 换淘宝镜像:
   ```powershell
   npm config set registry https://registry.npmmirror.com
   ```
2. 删掉重装:
   ```powershell
   Remove-Item -Recurse -Force node_modules, package-lock.json
   npm install
   ```

**Q: `npm install` 报 EACCES 权限错误**
A: 不要用管理员 PowerShell 跑 npm; 项目目录不要放 `C:\Program Files\`。

**Q: 我修改了代码, 但浏览器没变化**
A: 前后端都有热更新, 应该自动刷新。如果没刷新, 浏览器强刷 (Ctrl+Shift+R)。

---

### FAQ 3 · 数据库相关

**Q: `01-reset-db.ps1` 报 "无法加载文件 ... 在此系统上禁止运行脚本"**
A:
```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
# 输 Y 回车
```
然后重新跑脚本。

**Q: `01-reset-db.ps1` 中途报错停了**
A: 把错误信息拉到最上面看第一行错。常见:
- `password authentication failed` → `$env:PGPASSWORD` 没设对
- `database "booknook" is being accessed by other users` → 后端在连接, 先 Ctrl+C 停后端再 reset

**Q: 我想看数据库里有什么**
A: 用 pgAdmin (装 PG 时附带) 或命令行:
```powershell
psql -h 127.0.0.1 -U postgres -d booknook
# 进入后:
\dt           # 看所有表
SELECT * FROM books LIMIT 5;
\q            # 退出
```

---

### FAQ 4 · 后端启动相关

**Q: 启动报 `Invalid environment variables: JWT_SECRET 不能使用模板/示例默认值`**
A: 回 §4.3.2 重新生成。

**Q: 启动报 `Can't reach database server at localhost:5432`**
A: PostgreSQL 服务没启动:
```powershell
Start-Service postgresql-x64-16
```

**Q: 启动报 `Authentication failed against database server`**
A: `.env` 里的 `DATABASE_URL` 密码不对。检查这一行:
```
DATABASE_URL=postgresql://postgres:你的密码@127.0.0.1:5432/booknook?schema=public
```

如果密码里有特殊字符 (`@/?#` 等):
- `@` → `%40`
- `/` → `%2F`
- `#` → `%23`
- `?` → `%3F`

**Q: 端口 4000 被占用 `EADDRINUSE`**
A:
1. 找占用进程:
   ```powershell
   netstat -ano | findstr :4000
   ```
   看最后一列的 PID, 然后:
   ```powershell
   taskkill /PID <PID> /F
   ```
2. 或者改 `backend\.env` 的 `PORT=4001`, 同时改 `frontend\next.config.mjs` 的代理目标。

---

### FAQ 5 · 前端相关

**Q: 浏览器打开是白屏, F12 控制台一堆红字**
A: 看具体红字:
- `Failed to fetch /api/auth/me` → 后端没起来, 看 [窗口 B]
- `Hydration failed` → 强刷 (Ctrl+Shift+R)

**Q: 登录后立刻被踢回登录页**
A: cookie 过期或 JWT_SECRET 被换过。打开 F12 → Application → Cookies → 删除 `booknook_token` → 重新登录。

**Q: 看到的页面是英文的**
A: 之前切到了英文, 顶栏 Languages 图标点回中文。

**Q: 我能不能用手机访问?**
A: 默认 `localhost` 只能本机访问。手机访问需要:
1. 电脑和手机在同一 WiFi
2. 看 [窗口 C] 启动时打印的 `Network: http://192.168.x.x:3000`
3. 手机浏览器输入这个地址
4. 后端也要改 CORS_ORIGINS 加上手机地址

| **依赖 (dependency)** | 项目用到的第三方库, 列在 `package.json` 里 |
| **后端 (Backend)** | 在服务器上跑的程序, 处理业务逻辑 + 鉴权 + 数据库 |
| **前端 (Frontend)** | 在浏览器里跑的程序, 显示页面 |
| **API** | 后端给前端 (或其他客户端) 提供的"接口", 比如 `POST /api/login` |
| **JWT** | 一种"通行证"格式, 用户登录后后端发一个, 后续请求带着这个证明身份 |
| **Cookie** | 浏览器和服务器之间传递的小段数据, 常用来存登录态 |
| **HttpOnly Cookie** | 一种特殊的 Cookie, JavaScript 读不到, 只有浏览器自动带给后端, 防 XSS |
| **CORS** | 浏览器的安全机制, 限制网页和不同域名服务器交互。后端要明确允许哪些前端地址 |
| **PostgreSQL** | 一种关系型数据库 (RDBMS), 类似 MySQL 但功能更强 |
| **psql** | PostgreSQL 自带的命令行客户端 |
| **Prisma** | 后端用的"翻译官": 把 JavaScript/TypeScript 代码翻译成 SQL 跑数据库 |
| **Schema** | 数据库的"结构定义", 哪些表、每张表有哪些字段 |
| **Migration / Seed** | Migration = 数据库结构升级脚本; Seed = 灌初始数据 |
| **Trigger (触发器)** | 数据库里的"小程序", 在特定操作 (INSERT/UPDATE/DELETE) 时自动跑 |
| **MD5** | 一种哈希算法, 把任意输入变成固定 32 位十六进制字符串。本项目用于密码哈希 (加 salt) |
| **Salt (盐)** | 给密码额外加的随机字符串, 即使两人密码相同, 哈希结果也不同, 防彩虹表 |
| **环境变量 (.env)** | 项目配置, 比如数据库密码, 写在 `.env` 文件里不进入代码仓库 |
