## 一、启动（三个 PowerShell 窗口，按序执行）

```powershell
# 窗口 A · 数据库（一次性，看到 [DONE] 即可关闭此窗口）
cd E:\Github\Booknook
$env:PGPASSWORD = "你的postgres密码"
.\scripts\00-init-db.ps1

# 窗口 B · 后端（保持开着，等到 [OK] ... ready on http://localhost:4000）
cd E:\Github\Booknook
.\scripts\10-start-backend.ps1

# 窗口 C · 前端（保持开着，等到 ✓ Ready）
cd E:\Github\Booknook
.\scripts\11-start-frontend.ps1
```

**手动等价**（不想用脚本时）：
- 后端：`cd backend` → `npm install`（首次）→ `npx prisma generate`（首次）→ `npm run dev`
- 前端：`cd frontend` → `npm install`（首次）→ `npm run dev`

**验证**：`curl.exe http://localhost:4000/health` 返回 `{"ok":true}`；浏览器打开 **http://localhost:3000**。

**停止**：前端、后端窗口各按 `Ctrl + C` → 输 `Y`。PostgreSQL 是后台服务，不用停。
**重置数据**：`$env:PGPASSWORD="你的postgres密码"` → `.\scripts\01-reset-db.ps1`（含演示数据）或 `.\scripts\02-reset-empty.ps1`（空白态）。

## 二、登录

| 用户名 | 密码 | 角色 |
|---|---|---|
| `super` | `Admin@2026` | 超级管理员（可见全部 10 菜单） |
| `admin1` / `admin2` | `Admin@2026` | 普通管理员（无「用户管理」「操作日志」） |

## 三、11 项基础功能（最短演示步骤）

**1. 用户管理** `/users`（超管）
- 改自己资料：头像 → `[个人信息]` → 改姓名/性别/年龄 → `[保存]`；改密码：输旧密码 + 新密码（≥6 位）。
- 新建用户：`[新建用户]` → 填用户名/密码/姓名/工号/性别/角色 → `[保存]`。
- 停用/启用：列表右侧 `[停用]`/`[启用]`（不能停用自己，不能停用最后一个超管）。

**2-3. 库存查询** `/books`
- 左上下拉选字段（全部/编号/ISBN/书名/作者/出版社）+ 右侧输关键词回车。
- 样例：作者搜 `尤瓦尔` → 命中 2 本；ISBN 搜 `9787020002207` → 《红楼梦》。库存徽章：绿=充足 / 琥珀=告急 / 红=缺货。

**4. 库存修改** `/books`
- 点某行 `[✏️]` → 改书名/作者/出版社/零售价/分类/库存预警阈值 → `[保存]`，列表即刷新。

**（附）库存删除** `/books`
- 点某行 `[🗑️]` 删除（库存>0 或已有销售记录时拒绝）。

**5. 创建进货单** `/purchases`
- `[+ 新建进货单]` → 填供应商；已有书填 ISBN 失焦自动带出书名 → 填进货价+数量；`[+ 增加一行]` 录新书（ISBN/书名/出版社/作者/进货价/数量）→ `[提交进货单]`，状态 `未付款`。

**6. 进货付款** `/purchases`
- 未付款单点 `[付款]` → 确认 → 状态变 `已付款`，`/transactions` 多一条支出流水。

**7. 进货退货** `/purchases`
- 未付款单点 `[退货]` → 状态变 `已退货`（已付款的单子不可退，按钮不显示）。

**8. 进货入库** `/purchases`
- 已付款单点 `[入库]` → 弹对话框：新书必填零售价（默认预填进货价×1.5），老书只读 → `[确认入库]` → 状态变 `已入库`；`/books` 中老书库存+数量、新书自动上架。

**9. 销售收银** `/sales`
- 搜书 → 点 `[+ 加入]` 进购物车 → 调数量/填客户备注 → `[结账]` → toast `结账成功 SO-...`；`/books` 库存减、`/transactions` 增收入流水。

**10. 财务管理** `/transactions`
- 顶部 3 张汇总卡：收入合计/支出合计/净收益；列表每条标 `+`收入 / `−`支出，并关联对应销售单/进货单。

**11. 查看账单** `/transactions`
- Tab：全部/收入/支出；日期范围：起始 + 截止；`[重置]` 清空筛选。样例：Tab 切支出 + 选日期段 → 只剩进货付款记录。

## 四、7 项加分项（最短演示步骤）

**1. 数据可视化 Dashboard** `/dashboard`
- 4 张 KPI 卡（今日/本月销售、总库存、待付进货）+ 近 30 天销售趋势 + Top10 畅销书 + 月度收支双柱 + 库存分类饼图 + 库存预警列表。

**2. 库存预警** `/alerts`
- 造预警：`/books` 编辑某书把「库存预警阈值」改 `9999` → 保存 → 顶栏铃铛出现红点+脉冲 → `/alerts` 出现该书。
- 解除：阈值改回 `5`（或进货入库使库存回升）→ 预警自动消失。

**3. 操作日志审计** `/logs`（超管）
- 查看全部写操作；按动作/资源筛选；点某行展开看 JSON payload；改书价那条含 `before/after` 快照对比。

**4. 报表导出** `/reports`
- 4 张卡：销售/进货/财务 XLSX + 财务 PDF。点 `[导出]` 下载 → XLSX 中文表头+合计行，PDF 含汇总头+明细。

**5. 国际化（中英切换）** 顶栏语言图标 → 整站中⇄英瞬间切换，刷新页面保留选择。

**6. 暗色模式** 顶栏月亮/太阳图标 → 全站平滑切换深/浅色主题。

**7. RBAC 权限隔离** 退出 → 登录 `admin1`
- 侧栏「用户管理」「操作日志」消失；地址栏手输 `/users` 自动弹回 dashboard；后端 `curl /api/users`（带 admin1 cookie）返回 `403`。

## 五、10 分钟体验顺序（速记串场）

登录 → Dashboard 概览 → `/books` 查询+改价 → `/purchases` 建单 → 付款 → 入库 → `/sales` 结账（故意把数量改 999 触发「库存不足」高亮）→ `/alerts` 造预警再解除 → `/reports` 导出 XLSX/PDF → `/logs` 看 before/after → 切 `admin1` 验 RBAC → 顶栏切英文 + 暗色 → 退回 super 收工。

> 涵盖 11 项基础功能 + 7 项加分项 + 库存不足等边界保护。更深入的原理与 FAQ 见 **操作手册.md**。
